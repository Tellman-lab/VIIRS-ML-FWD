# VIIRS-ML-FWD

Machine Learning Fractional Water Detection with VIIRS Satellite Imagery

## About

This repository contains code to run VIIRS ML-FWD, a satellite water detection algorithm designed for global flood monitoring. The model ingests imagery from NASA's Visible Infrared Imaging Radiometer Suite (VIIRS) sensor aboard the Suomi NPP and NOAA-20 satellites to generate per-pixel estimates of fractional water extent (a continuous value between 0 and 1) at approximately 375 m resolution. The repository contains instructions and code to obtain the VIIRS data and run the pre-trained global algorithm to create inundation estimates.

## Project Structure

```
├── Inference/           # Code to run inference
│ ├── run_inference.sh   # Shell script to modify arguments and run the inference script
│ ├── Inference.py       # Main inference python script
│ └── Helpers            # Folder containing helper functions for inference
├── *Data/               # Raw VIIRS data (e.g., LAADS-DAAC order downloads)
├── *Inputs/             # VIIRS input data in format required for inference
├── *Inferred/           # Output inundation estimates generated by the inference code
├── *ModelWeights/       # Folder containing the model weights file for inference
├── .gitignore           # Files/folders to ignore in git
├── requirements.txt     # Python dependencies
└── README.md            # This file containing description
```

__Note:__ folders in the project structure marked with a leading asterisk are ignored by git and will initially not be present. These folders will be created by the user when using the code.

## Setup

1) __Clone repository__: ```git clone https://github.com/Tellman-lab/VIIRS-ML-FWD.git``` and navigate to the repository ```cd VIIRS-ML-FWD```.
2) __Create an environment and install Python dependencies__: ```conda env create -f environment.yml``` will create a new environment called ```viirs-ml-fwd```.
3) __Download the model weights file:__ from the [Releases](https://github.com/Tellman-lab/VIIRSGlobalFlood/releases/tag/v1.0.0) page and save the file ```model.pth``` in a folder named ```ModelWeights``` in the root path of this repository.
4) __Create a NASA Earth Data [account](https://urs.earthdata.nasa.gov/profile):__ to be able to download VIIRS scenes from NASA LAADS-DAAC.
   
## Description and Usage

### Download Data

__Order the VIIRS scenes from the [LAADS-DAAC web portal](https://ladsweb.modaps.eosdis.nasa.gov/search/):__ Follow these steps to search for and order VIIRS data from LAADS-DAAC. The pre-processing of the VIIRS scenes is handled on the LAADS DAAC server and the output is a composite VIIRS scene ready for the inference pipeline.
   
   * __Product:__ search for and select the product 'VNP09' (collection 5200), VIIRS imgary from the Suomi NPP satellite. You may also use the product 'VJ109' (collection 5200) to use VIIRS imagery from the NOAA-20 satellite (note that SNPP and NOAA-20 have different overpass times so the scenes will not be identical).
   * __Time:__ enter a single date or date window.
   * __Location:__ enter a region of interest. LAADS-DAAC offers various options inlcuding drawing a box, entering corner coordinates, searching for a country or administrative area.
   * __Files:__ select all files.
   * __Review & order:__ select the 'Apply Post-Processing' drop down. Sds: select bands I1, I2, I3, M3, M4, M11, QF1, and QF2. Geo: check apply from search criteria. Mosaic: check geographically align. Reformat: select geoTIFF. Reproject: select Mercator. Submit order.
   * __Navigate to [Past Orders](https://ladsweb.modaps.eosdis.nasa.gov/search/history):__ to find the order number for the order you submitted. You will need this order number for the inference pipeline. When the data is ready, the order will show the status 'Available'. This can take as little as several minutes or as long as several days depending on the size of the order and other traffic on the web portal. Note that orders automatically expire after a period of 30 days.

### Inference

__Steps:__

The __[Inference](https://github.com/Tellman-lab/VIIRSGlobalFlood/blob/main/Inference) code__ performs several steps, each of which can optionally be omitted:

1) __Get the LAADS-DAAC VIIRS order:__ automatically extract the pre-processed VIIRS scenes from LAADS-DAAC using the order number you created during the download step and your login credentials (email and API key token). The code uses a simple wget command to extract the data, one .tif file per band. For this option to work, make sure you followed all of the steps described above under Download Data.
2) __Resample the VIIRS data:__ to exactly 375 m and 750 m spatial resolution as required by the inference model. The reprojected Mercator outputs from LAADS-DAAC are close but not exactly the spatial resolutions. This part performs a simple resampling to ensure the exact specifications are met for inference.
3) __Prepare the band input files:__ create input .tif files in the format (i.e. folder and naming convention) required for inference. Perform a check to make sure all the required files are present before attempting inference. This is performed separately for every date and granule (i.e., every date-time) in your VIIRS inputs. From hereon, all processing is performed separately for each date and granule (i.e., every unique date-time) and data are created in sub folders named with the corresponding date (in yyyydoy format) and time. Where there is only one file for a given date, the time naming is omitted.
4) __Chip the band inputs:__ chip the input data into patches of size 256 x 256 pixels as required by the inference model. As in step (3), this is performed separately for every date and granule (i.e., every date-time) in your VIIRS inputs.
5) __Run inference:__ load the model weights and run the inference model on the chipped band inputs. Creates a chipped .tif raster file containing continous values between 0 and 1 for every pixel.
6) __Mosaic inferred chips:__ spatially merge the inferred chips to create a single inferred .tif raster output for each unique date-time.

__Folders created:__

* __Data:__ the 'raw' VIIRS data downloaded from the LAADS-DAAC web portal. There must be one .tif file for each band including the band name within the file name (i.e., I1, I2, I3, M3, M4, M11, QF1), in Mercator projection.
* __Inputs:__ the VIIRS input data for model inference. Contains a copy of each band file from Data with correct naming convention (the band name), then the chipped version of each band file at 256 x 256 pixel resolution.
* __Inferred:__ the inferred outputs containing continous values between 0 and 1 for every pixel.

__Run the inference code:__

Navigate into the folder ```cd Inference```, activate your environment e.g., ```mamba activate viirs-ml-fwd```, and run ```./run_inference.sh```. 

__Arguments:__

Arguments should be modified in the file ```run_inference.sh```. For a complete description of the arguments, navigate into the folder ```cd Inference```, activate your environment e.g., ```mamba activate viirs-ml-fwd```, and run ```python Inference.py --help```. 

The only *required* argument is the ```---folderName```. This is the name of the folder containing the VIIRS data and must be the VIIRS order number generated automatically when you submitted your data download order via LAADS-DAAC. You can obtain this order number by navigating to [Past Orders](https://ladsweb.modaps.eosdis.nasa.gov/search/history). 

You may provide one or multiple folder names. Simply provide them as a sequence of strings separated with blank spaces, e.g. ```502416879 502416901```. The inference pipeline will run in sequence for each folder name provided.

Many of the optional arguments allow you to either include or omit steps of the pipeline described above depending on your needs. Under the default settings, all steps will be run and any existing files for the corresponding data folder (i.e., VIIRS order) will be overwritten. However, if for example you have already downloaded, prepared the VIIRS inputs, and created chips, you may choose to omit all stages prior to the inference step. To do so, you would set the relevant arguments to ```n``` in the file ```run_inference.sh```.

A few __specific arguments__ are worthy of note:

* ```--maskClouds```: you may set cloud-obscured pixels in the output to no data value (-28672). This uses the default cloud mask in the raw VIIRS product. The default option is to keep predictions for all pixels. We recommend this because (a) the VIIRS cloud mask is known to be conservative (often an overestimate of cloud cover and can falsely identify water as clouds), and (b) the inference model is sometimes able to generate reliable predictions despite cloud cover thanks to the use of the SWIR bands which penetrate some thin clouds.  
* ```--imageBuffer``` and ```--gradientMethod```: these arguments control the feathering process applied to the inference. Feathering helps to reduce artifact predictions at the edge of inferred chips where some conditions (mostly cloud-cover) can cause artificial discontinuities. Feathering creates overlapping chips for inference and applies a weighting mask to carefully average predictions from multiple chips, acting somewhat like a majority voting system. By default, an image buffer of 96 pixels is applied which we found to be a reasonable tradeoff between effective feathering and computational efficiency.

__Warning about regions that cross the 180 longitude line:__ 

If your region of interest spans the 180 longitude line, split your region into two parts and run the process separately for each part. For example, perform your data download (regardless of the method chosen) for two separate regions of interest. Then follow the rest of the process as usual, treating the two regions as separate folders for inference.
